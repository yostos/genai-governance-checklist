# 生成AI利用ガイドライン サンプル

本フォルダには、
「生成AI利用ガイドラインチェックリスト」のRequired / Recommended項目を
満たすガイドラインのサンプル文書を収録しています。

## 概要

このサンプルは、専門IT部門を持たない組織が
生成AI利用ガイドラインを策定する際の参考文書です。
チェックリストの各項目に対応する具体的な記述例を示しています。

## ファイル構成

| ファイル | 内容 |
|----------|------|
| `organization-profile.md` | 想定組織のプロファイル（架空） |
| `requirements-for-guideline.md` | ガイドラインの要件 |
| `guideline-sample.md` | 生成AI利用ガイドライン（サンプル本文） |
| `guideline-sample.html` | 上記のHTML版 |
| `Makefile` | HTML生成用 |

## ビルド方法

```bash
make        # HTMLを生成
make clean  # 生成物を削除
```

pandocが必要です。

## 想定組織

- **組織名**: 一般社団法人中小企業経営支援センター（架空）
- **規模**: 職員233名（常勤33名、嘱託職員200名）、専門IT部門なし
- **業務**: 中小企業向け経営相談
- **特徴**: 顧客の機密情報・個人情報を日常的に取り扱う

詳細は `organization-profile.md` を参照してください。

## 対応するチェックリスト項目

| レベル | 対応状況 |
|--------|----------|
| Required | 74項目すべて |
| Recommended | 重要度の高い19項目 |
| Option | 対象外 |

各項目との対応関係は、サンプル本文中に `[対応: 1.1.A]` 等の形式で注記しています。

## 生成AIサービスの前提条件

本ガイドラインは、ChatGPT / Claude / Gemini 等の
組織向けプラン（Team/Business/Enterprise等）が
選定されることを前提としています。
個人向けプラン（Free/Pro等）は、
管理者機能やデータ保護の面で組織利用の要件を満たしません。

組織向けプランの導入にあたっては、以下の運用を確実に行ってください。

- 選定したプランで入力データがモデルの学習に使用されないことを確認する
- サービス提供者のデータ保持・削除ポリシーを確認し、記録しておく
- アカウントの発行・無効化を管理者が一元的に管理する
- 入退職時のアカウント処理手順を定め、退職者のアカウントを速やかに無効化する
- サービス提供者の利用規約・プライバシーポリシーの変更を定期的に確認する

## 利用上の注意

- **本サンプルの想定組織は、IT Governance の基盤が十分でない状態で
  生成AIを導入するケースです。**
  情報セキュリティポリシーの運用実態が不十分、
  CIO不在、専門IT部門なしという前提の下で、
  「まず最低限のルールを整備する」ことを優先した実用重視の内容です。
  理想的なガイドラインではなく、現実的な出発点として位置づけてください。
  本来は、生成AIの導入に先立ち、
  IT Governance 体制そのものを整備することが望ましいです。
- 本サンプルはあくまで参考例です。
  実際のガイドライン策定時は、自組織の状況に合わせて内容を調整してください。
- Optionレベルの項目は含まれていません。
  必要に応じて追加を検討してください。
- 法令・規制は変更される可能性があります。
  最新の情報を確認の上、利用してください。
- 組織の職員のITリテラシーに応じて、
  FAQ（よくある質問）セクションを作成し、
  ガイドラインに追加することを推奨します。
  運用開始後に寄せられた質問を蓄積し、
  ガイドライン改訂時に反映するとよいでしょう。
