# ガイドラインサンプルの要件

## 基本方針

初回策定のガイドラインとして、最低限の実効性と体裁を確保する。

## 想定読者

- 非エンジニア
- ITリテラシーが最低レベルの職員
- 「生成AI」という言葉は聞いたことがある程度

専門用語は避け、平易な日本語で説明する。
なぜそのルールが必要なのか、背景や理由も含めて記述する。

## チェックリスト対応

| レベル | 対応方針 |
|--------|----------|
| Required | すべて満たす |
| Recommended | 重要度の高いものを選定して満たす |
| Option | 見送り（将来の改訂で検討） |

## 文書としての体裁

組織標準文書として最低限必要な要素を備える：

- バージョン番号
- 制定日・施行日
- 承認者
- 改訂履歴
- 目次
