# 想定組織プロファイル

ガイドラインサンプルの対象となる架空の組織を定義する。

## 基本情報

| 項目     | 内容                                         |
| -------- | -------------------------------------------- |
| 組織名   | 一般社団法人 中小企業経営支援センター        |
| 設立     | 2006年4月                                    |
| 所在地   | 東京都中央区                                 |
| 職員数   | 233名（常勤33名、嘱託職員200名）             |
| 事業内容 | 中小企業向け経営相談、創業支援、事業承継支援 |

## 組織の特徴

### 規模・体制

- CIOが存在しない。IT Governanceの基本的な体制もない。
- 専門のIT部門は存在しない
- 総務担当者が情報システム管理を兼務
- 顧問弁護士・顧問税理士と契約あり

### 業務の性質

- 中小企業経営者からの経営相談が主業務
- 相談内容には顧客企業の財務情報、事業戦略、人事情報、個人情報が含まれる
- 守秘義務を伴う情報を日常的に取り扱う
- 行政機関からの委託事業も受託（補助金申請支援等）

### 情報セキュリティ環境

- 情報セキュリティポリシー策定済みだが、自治体から拝借しただけで運用されているか疑問。セキュリティ責任者は機能していない。
- 個人情報保護方針策定済み。
- クラウドサービス（Microsoft 365）を業務基盤として利用
- 顧客情報は自社作成システムによりオンプレミスで管理

## AI利用の現状と期待

### 現状

- 一部職員が個人的にChatGPT等を試用している
- 組織としての利用ルールは未整備
- AI利用の可否について職員から問い合わせが増加
- 理事長がIT Governanceもできていないのに「生成AIを導入する」と打ち上げた。

### 期待する活用場面

1. **文書作成支援**: 報告書・提案書の草稿作成
2. **調査補助**: 業界動向・制度変更の情報収集
3. **翻訳・要約**: 英文資料の翻訳、長文資料の要約
4. **アイデア出し**: 経営改善策の検討補助

### 懸念事項

- 顧客の機密情報、個人の機微情報がAIサービス経由で漏えいするリスク
- AIが生成した誤情報を顧客に提供してしまうリスク
- 職員間で利用ルールの認識がばらつくリスク
- 著作権侵害のリスク

## ガイドライン策定の前提条件

### 利用予定の AI サービス

- ChatGPT Team（OpenAI）/ Claude (Anthropic) / Gemini (Google) などのチャットベースの生成AI

いずれも海外サーバーでデータ処理されるSaaS型サービス。
管理者向けのログ閲覧機能は限定的。

### 制約事項

1. **個人データの入力禁止**: 海外サーバー処理のため、
   個人データの入力は原則禁止とする
2. **モニタリングの限界**: 会話内容レベルの監視は技術的に困難、
   利用者の自己申告と啓発に依存
3. **リソースの制約**: 専任のAI管理者を置く余裕はなく、
   総務担当者が兼務

### 既存規程との関係

- 情報セキュリティポリシー: 外部クラウドへの機密情報送信を制限
- 個人情報保護方針: 第三者提供・海外移転に関する規定あり
- 就業規則: 懲戒規定あり

生成AI利用ガイドラインは、これら既存規程と整合させる。

## 組織図（簡略）

```
理事長（最終責任者）
    │
    ├── 事務局長
    │       └── 総務部 ── IT管理担当者を兼務
    │
    ├── 相談支援部長
    │       └── 相談支援部
    │
    └── 事業推進部長
            └── 事業推進部

```

## 想定する関係者

| 役割             | 担当者           | 備考                                   |
| ---------------- | ---------------- | -------------------------------------- |
| AI利用最終責任者 | 理事長           | 方針承認、重大インシデント時の意思決定 |
| AI管理責任者     | 総務部長         | 財源確保、成果責任、サービス選定責任   |
| AI管理担当者     | 総務担当者       | 日常管理、問い合わせ対応、研修企画     |
| 利用者           | 全職員           | 業務での AI 利用者                     |
| 外部専門家       | ITコンサルタント | 技術的助言、ツール選定支援             |
| 外部専門家       | 顧問弁護士       | 法的判断、契約確認                     |
